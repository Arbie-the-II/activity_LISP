

# CLI Space Invaders in Common Lisp

A real-time, lightweight, zero-dependency Space Invaders clone written in Common Lisp. This game runs natively inside the Windows Command Prompt or PowerShell using direct Win32 API console polling for non-blocking keyboard inputs.



## 🎮 Game Controls

| Key | Action |
| :--- | :--- |
| **`A`** | Move Spaceship Left |
| **`D`** | Move Spaceship Right |
| **`Spacebar`** | Fire Laser Canon |
| **`Q`** | Quit Game instantly |



## 🏗️ Project Architecture

The game is separated into two micro-files to isolate the compilation parameters from the active game loops:

1. **`game.lisp`**: Manages entity coordinates, collision detection vector math, low-level keyboard polling, and frame rendering.
2. **`build-game.lisp`**: Instructs the Steel Bank Common Lisp (SBCL) compiler to strip background runtime threads, compile everything to native machine code, and save it as a standalone executable.

## 💻 Step-by-Step Installation & Build

Follow these exact steps inside your Windows PowerShell terminal to set up and compile the game:

### Step 1: Initialize Workspace Directory
Navigate to your desired programming folder and create a dedicated location for your project

### Step 2: Write the Compilation Script (`build-game.lisp`)

Create the setup script using your text editor or IDE and save it as `build-game.lisp`. This handles the compiler core configuration safely:

```lisp
;; 1. Load Quicklisp environmental setups
(let ((quicklisp-init (merge-pathnames "quicklisp/setup.lisp" (user-homedir-pathname))))
  (when (probe-file quicklisp-init)
    (load quicklisp-init)))

;; 2. Load the main game logic code
(load "game.lisp")
(in-package :cl-user)

;; 3. Terminate background threads before freezing core memory
(dolist (thread (sb-thread:list-all-threads))
  (unless (eq thread sb-thread:*current-thread*)
    (ignore-errors (sb-thread:terminate-thread thread))))

;; 4. Freeze memory image into a standalone binary console executable
(sb-ext:save-lisp-and-die "space-invaders.exe" 
                          :toplevel (intern "START-GAME" "SPACE-GAME")
                          :executable t 
                          :compression nil
                          :application-type :console)

```

### Step 3: Write the Game Engine (`game.lisp`)

Create a new text file named `game.lisp` and add the pure engine code. We use standard ASCII elements (`A` for player, `V` for invaders, `|` for lasers) to ensure 100% rendering compatibility across older and newer Windows terminals alike:

```lisp
(defpackage :space-game
  (:use :cl)
  (:export :start-game))

(in-package :space-game)

;; --- CONSTANTS & CONFIGURATION ---
(defparameter *width* 40)
(defparameter *height* 20)

;; --- GAME STATE ---
(defparameter *player-x* 20)
(defparameter *bullets* nil)
(defparameter *invaders* nil)
(defparameter *invader-dir* 1)
(defparameter *score* 0)
(defparameter *game-over* nil)

;; --- LOW-LEVEL WINDOWS INPUT HOOKS (NON-BLOCKING) ---
#+sbcl
(sb-alien:define-alien-routine ("_kbhit" kbhit) sb-alien:int)
#+sbcl
(sb-alien:define-alien-routine ("_getch" getch) sb-alien:int)

(defun read-keyboard-input ()
  #+sbcl
  (if (/= (kbhit) 0)
      (let ((ch (getch)))
        (cond ((= ch 97)  :left)
              ((= ch 100) :right)
              ((= ch 32)  :space)
              ((= ch 113) :quit)
              (t nil)))
      nil)
  #-sbcl nil)

;; --- GAME LOGIC ENGINE ---

(defun init-game ()
  (setf *player-x* 20
        *bullets* nil
        *score* 0
        *game-over* nil
        *invader-dir* 1
        *invaders* nil)
  (dotimes (row 3)
    (dotimes (col 6)
      (push (cons (+ 10 (* col 4)) (+ 2 row)) *invaders*))))

(defun update-game ()
  ;; 1. Move bullets up
  (setf *bullets* (delete-if (lambda (b) (<= (cdr b) 0))
                             (mapcar (lambda (b) (cons (car b) (1- (cdr b)))) *bullets*)))
  ;; 2. Move invaders
  (let ((shift-down nil)
        (move-tick (zerop (mod (get-internal-real-time) 4))))
    (when move-tick
      (dolist (inv *invaders*)
        (let ((next-x (+ (car inv) *invader-dir*)))
          (when (or (<= next-x 0) (>= next-x (1- *width*)))
            (setf shift-down t))))
      (if shift-down
          (progn
            (setf *invader-dir* (- *invader-dir*))
            (setf *invaders* (mapcar (lambda (inv) (cons (car inv) (1+ (cdr inv)))) *invaders*)))
          (setf *invaders* (mapcar (lambda (inv) (cons (+ (car inv) *invader-dir*) (cdr inv))) *invaders*)))))
  ;; 3. Collision handling
  (dolist (bullet *bullets*)
    (let ((hit (find-if (lambda (inv) (and (= (car bullet) (car inv)) (= (cdr bullet) (cdr inv)))) *invaders*)))
      (when hit
        (setf *invaders* (remove hit *invaders*))
        (setf *bullets* (remove bullet *bullets*))
        (incf *score* 100))))
  ;; 4. Check status
  (dolist (inv *invaders*)
    (when (>= (cdr inv) (1- *height*))
      (setf *game-over* t)))
  (unless *invaders*
    (setf *game-over* :win)))

;; --- CLI RENDERING ENGINE ---

(defun clear-screen ()
  (format t "~C[H" #\Esc)
  (finish-output))

(defun render-frame ()
  (clear-screen)
  (format t "==========================================~%")
  (format t " SCORE: ~,5D       Controls: A=Left, D=Right, Space=Fire, Q=Quit~%" *score*)
  (format t "==========================================~%")
  (dotimes (y *height*)
    (format t "|")
    (dotimes (x *width*)
      (cond
        ((and (= y (1- *height*)) (= x *player-x*)) (format t "A"))
        ((member (cons x y) *bullets* :test #'equal) (format t "|"))
        ((member (cons x y) *invaders* :test #'equal) (format t "V"))
        (t (format t " "))))
    (format t "|~%"))
  (format t "==========================================~%")
  (finish-output))

;; --- MAIN RUNTIME GAME LOOP ---

(defun start-game ()
  (format t "~C[2J" #\Esc)
  (init-game)
  (loop
    (let ((input (read-keyboard-input)))
      (case input
        (:left  (setf *player-x* (max 0 (1- *player-x*))))
        (:right (setf *player-x* (min (1- *width*) (1+ *player-x*))))
        (:space (push (cons *player-x* (- *height* 2)) *bullets*))
        (:quit  (return)))
      (update-game)
      (render-frame)
      (cond 
        ((eq *game-over* t)
         (format t "~%GAME OVER! The invaders reached Earth.~%")
         (return))
        ((eq *game-over* :win)
         (format t "~%VICTORY! You saved the galaxy!~%")
         (return)))
      (sleep 0.03))))

```

### Step 4: Run the Compiler Compilation Loop

Trigger the native compilation system within your PowerShell shell:

```powershell
sbcl --no-userinit --load build-game.lisp

```

*Note: This generates a standalone executable binary file called `space-invaders.exe` in the current folder.*

### Step 5: Launch and Play

Boot up the compiled game binary application to start playing right away:

```powershell
.\space-invaders.exe

```

## 🛠️ Code Deep Dive & Design Patterns

### 1. Dynamic Symbol Interning

To prevent the Common Lisp compiler reader from panicking about packages that haven't been completely parsed into the memory layout yet, the compilation script leverages dynamic interning:

```lisp
(intern "START-GAME" "SPACE-GAME")

```

This tells the reader to look up and register the entry-point method dynamically at build time rather than early scanning routines.

### 2. High-Performance Frame Refresh

Instead of flashing using complete terminal buffer rewrites (`~C[2J`), which causes screen flickering, the code implements a localized positioning sequence reset:

```lisp
(format t "~C[H" #\Esc)

```

This forces the terminal cursor position to return straight back to home position coordinates `(0,0)` instantly, smoothly overwriting the current layout line by line.
